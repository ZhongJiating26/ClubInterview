# 引言

## 文档目的

本《数据库设计文档》用于定义校园社团招新与面试管理系统的数据库结构，包括数据模型、表结构、主外键关系、约束、索引、事务设计和存储策略等。

本文档为后端开发、测试人员、数据库管理员提供统一参考依据。

## 适用读者

- 后端开发工程师

- 前端开发工程师

- 测试工程师

- 项目负责人

- 数据库运维（DBA）

## 术语说明

| 术语     | 含义                                     |
|----------|------------------------------------------|
| RBAC     | 基于角色的访问控制                       |
| PK       | 主键                                     |
| FK       | 外键                                     |
| CRUD     | 增删改查操作                             |
| JSON字段 | MySQL JSON类型，用于动态表单、扩展字段等 |
| 软删除   | 使用is_deleted字段标记删除               |

# 数据库总体设计

## 数据库选型

- 数据库：MySQL 8.0+

- 引擎：InnoDB（支持事务、行锁、外键、崩溃恢复）

- 编码格式：utf8mb4

- 排序规则：utf8mb4_unicode_ci（支持多语言排序及 Emoji）

- 存储模式：业务与文件分离，附件文件保存在RustFS系统，数据库仅存文件URL

## 命名规范

### 表命名

使用snake_case小写+模块前缀

示例：

> user_account
>
> recruitement_session
>
> interview_record

### 字段命名

时间字段统一：created_at，updated_at

软删除字段：is_deleted TINYINT(1)、deleted_at DATETIME

### 索引命名

唯一索引：uniq_XXX

普通索引：idx_XXX

## 通用字段规范

所有业务表均包含以下字段,作用为支持审计、软删除、时间排序

| 字段名      | 类型       | 说明                               |
|-------------|------------|------------------------------------|
| created_at  | DATETIME   | 创建时间                           |
| updataed_at | DATETIME   | 更新时间                           |
| is_deleted  | TINYINT(1) | 逻辑删除标志（0:正常, 1:已删除）   |
| deleted_at  | DATETIME   | 删除时间（仅当is_deleted=1时有值） |

## 数据库总体结构与模块划分

数据库划分七大模块：

1.  用户与权限模块

2.  社团/部门/岗位模块

3.  报名模块

4.  面试管理模块

5.  通知中心模块

6.  FAQ/工单（在线答疑）

7.  系统日志模块

## 外键约束说明

本系统在数据库逻辑设计层面明确各表之间的关联关系，但在物理实现层面不强制使用数据库FOREIGN KEY约束。符合高并发互联网系统中“弱外键、强业务约束”的经验，便于未来拆分为微服务架构。

表间数据一致性由应用层逻辑保证，以降低数据库耦合度并提升系统扩展性与性能。

# 索引设计

## 索引设计原则

- 对常用查询字段建立联合索引

- 时间排序字段加入索引尾部

- 避免低选择性字段单独建立索引（如status）

- 所有列表接口一律分页（避免全表扫描）

- 所有列表默认按created_at DESC排序，将created_at放入索引尾部

# 数据库一致性与约束设计

## 主键约束

所有业务表均使用：

> BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY

## 唯一性约束

为支持“软删除”机制（is_deleted=1后允许重新插入相同业务键），本系统在数据库层使用MySQL 8.0函数索引实现“仅对未删除数据唯一”的约束，同时在Service层做兜底校验。

手机号唯一（user_account）

[TABLE]

同一用户在同一招新场次仅能有一个未删除报名（signup_session）

[TABLE]

同一模板内评分项排序号不重复（score_item）

[TABLE]

## 数据有效性约束

- 时间逻辑：start_time\<end_time

- 状态字段使用枚举，如PENDING, APPROVED, REJECTED

- 评分项表score_item使用CHECK约束保证权重和满分为正数：weight\>0，max_score\>0（MySQL 8.0）

# 事务设计

使用MySQL InnoDB事务，隔离级别：Repeatable Read（RR）

## 报名提交事务

包含：

- 创建报名主表

- 创建岗位子表

- 创建附件表（可选）

全部成功才提交，否则回滚。

## 面试评分事务

包含：

- 写入 interview_score明细

- 更新 interview_record的total_score和status

评分必须原子化，防止部分成功部分失败。

## 工单回复事务

包含：

- 写入 ticket_reply

- 更新 ticket.updated_at和status

- 密码：bcrypt哈希存储

- 手机号（脱敏显示）

is_deleted=1且deleted_at=NOW()
